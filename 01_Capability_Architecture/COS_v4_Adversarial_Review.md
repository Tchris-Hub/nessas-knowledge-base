# COS v4 Adversarial Review — Invalidation Attempt

**Objective:** Break the 8-function model (Frame, Explore, Select, Realize, Evaluate, Refine, Integrate, Communicate).
**Date:** June 15, 2026
**Method:** Adversarial. Do not defend. Attempt to destroy.

---

## 1. Whether These Eight Functions Are Truly Fundamental

**Verdict: No. They are a valid middle layer but NOT primitives.**

Evidence from eight creativity models synthesised shows that stage-based process models (Wallas, Geneplore) have been critiqued for decades as descriptive rather than explanatory. The 8-function model inherits this flaw — it describes what might happen but says nothing about *why* or *under which conditions* functions engage.

**The strongest counter-evidence:** Campbell's Blind Variation + Selective Retention (BVSR) model requires only TWO primitives: *generate variation* and *select*. All eight functions are refinements or compositions of these two. If 2 can explain what 8 claims to, 8 is overfit.

---

## 2. Whether Any Functions Are Missing

**Verdict: Yes. At least five significant phenomena are absent.**

| Missing Phenomenon | Research Support | Why It Matters |
|-------------------|-----------------|----------------|
| **Incubation** | Wallas (1926), Sio & Ormerod (2009) | Unconscious restructuring during breaks. Accounts for insight phenomena. None of the 8 functions are unconscious. |
| **Play** | Bateson (1955), Csikszentmihalyi (1996) | Non-goal-directed exploration. Generates variation without selection pressure. "Explore" is too goal-directed to capture this. |
| **Affect/Emotion** | Isen (1999, positive affect), Feist (1999), Kaufmann (2003) | Mood states fundamentally alter cognitive style. Positive affect broadens attention. Negative affect drives persistence. This is state-dependent, not function-invokable. |
| **Problem Finding** | Getzels (1975), Runco (1994) | Creativity often begins before "Frame" — with noticing an anomaly, sensing a gap, or identifying a problem that didn't previously exist. Frame assumes a problem is already given. |
| **Constraint Transformation** | Boden (1990, 2004), Stokes (2005) | The most creative acts often involve CHANGING the constraints, not operating within them. This is a meta-function above the other eight. |

---

## 3. Whether Some Functions Should Be Merged

**Verdict: Yes. At least two pairs collapse under scrutiny.**

### Evaluate + Refine: Inseparable Loop

In interactive-incremental models of creativity (Simon, 1973; Schön, 1983), evaluation is **continuous** — every partial realisation is immediately assessed and refined in a single loop. A jazz pianist does not "generate, then evaluate, then refine" — the generation IS the evaluation.

**Evidence from coding:** The REPL (Read-Eval-Print-Loop) demonstrates this perfectly. Write code → run → see output → adjust. There is no separation between "evaluate" and "refine." It's one microfunnel.

**Evidence from painting:** A painter applies a stroke, sees it, and adjusts the next stroke. The evaluation of the stroke and the refinement of the next stroke happen in the same perceptual-motor cycle.

### Realize + Evaluate + Refine: Triplet Collapse

In music improvisation, architecture sketching, and live coding: the act of making IS the act of judging. They cannot be separated temporally or cognitively. The triple collapses into what could be called **Generative Evaluation** — producing while assessing.

---

## 4. Whether Some Functions Are Emergent Rather Than Primary

**Verdict: Yes. Three functions show clear emergence.**

### Communication — Emergent from Social Context

Communication arises from the social/production need to share work with others, not from creativity itself. A creator working in private (sketchbook, personal journal, private composition) never needs Communication. It is an overlay function that activates only when the creative process enters a social context.

**Evidence:** Children's private speech during creative play, unpublished manuscripts, unfinished works. The creative act was complete without communication.

### Integration — Emergent from Complexity

Integration only becomes necessary when multiple elements exist that need to be composed. A single-element creation (a poem, a portrait, a melody) doesn't need Integration. It emerges as a function only at scale.

### Refinement — Emergent from Evaluation

Refinement is not a primary process. It is the behavioral expression of the Evaluate → Transform loop. You evaluate, you find a gap, you adjust. The "refinement" is just the observable output of this cycle repeating.

## 5. Cross-Domain Validity

**Verdict: The 8 functions work for SOME domains but break for others.**

| Domain | 8-Function Fit | Where It Breaks |
|--------|---------------|-----------------|
| **Film Production** | High — matches existing production pipeline | Misses: collaborative negotiation, budget constraints, team dynamics |
| **Music Composition** | Medium | Misses: embodied listening, improvisation (no separation of functions), emotional state, practice/repetition |
| **Architecture** | Medium-High | Misses: constraint optimization as PRIMARY driver, client negotiation, regulatory compliance, material physics |
| **Coding** | Low-Medium | REPL loop collapses Evaluate/Refine/Realize. Debugging is NOT "Refine" — it's hypothesis testing. Code reuse is NOT "Realize." |
| **Painting** | Low-Medium | Physical manipulation and sensory feedback dominate. The "dialogue with material" (Schön) is not captured. Serendipity and accident are features, not bugs. |
| **Writing** | Medium | Writer's block (failure state not modeled), drafting IS exploring, revision IS not the same as refinement (often complete rewrite). Non-linear composition. |
| **Inventing/Engineering** | Medium | The "Aha!" insight moment missing. Incubation plays a major role. Constraint transformation is the core activity. |
| **Improvisational Theatre** | Low | Real-time creation with NO separation between any functions. Generate, evaluate, and perform are simultaneous. The audience co-creates. |

**Conclusion:** The 8 functions over-generalize from a narrow set of cognitive-science examples (Finke's abstract shape experiments) and fail to account for domain-specific knowledge, tools, material practices, and social contexts.

## 6. Whether a Smaller Set of Primitives Exists Beneath These Eight

**Verdict: Yes. Irreducible primitives can be identified at 3-5.**

### Candidate 1: The 3-Primitive Model (Campbell/Simonton)

1. **Generate** — Produce novelty (blind variation)
2. **Select** — Retain viable variations against criteria
3. **Transform** — Modify existing forms

The 8 functions as compositions:

| Function | Composed From |
|----------|--------------|
| Frame | Generate(constraints) |
| Explore | Generate(many variations) + Select(loose criteria) |
| Select | Select(comparative, commit) |
| Realize | Generate + Transform + Select (continuous loop) |
| Evaluate | Select(against intent) |
| Refine | Select + Transform (iterative loop) |
| Integrate | Transform(combine) + Select(coherence) |
| Communicate | Transform(format) |

### Candidate 2: The 4-Primitive Model (Nessa — proposed)

Adds **Perceive/Attend** as a distinct primitive because attention direction is a separable cognitive act:

1. **Attend** — Direct cognitive resources. What matters?
2. **Generate** — Produce novelty. Idea, form, connection.
3. **Evaluate** — Judge against criteria. Implicit or explicit.
4. **Transform** — Change existing forms. Adjust, combine, restructure.

The 8 functions:
| Function | Composed From |
|----------|--------------|
| Frame | Attend(what matters?) + Generate(constraints) |
| Explore | Attend(broad) + Generate(many) + Evaluate(loose) |
| Select | Evaluate(comparative) + Attend(commit) |
| Realize | Attend(focus) + Generate + Evaluate + Transform (tight loop) |
| Evaluate | Evaluate(against intent) |
| Refine | Evaluate(loop) + Transform |
| Integrate | Transform(combine) + Evaluate(coherence) |
| Communicate | Transform(format) + Attend(audience) |

### Candidate 3: The 2-Primitive Model (Pure BVSR)

1. **Generate** — Blind variation
2. **Select** — Selective retention

**Strength:** Most parsimonious. Explains evolution, learning, and creativity with one mechanism.
**Weakness:** Too abstract to be useful for system design. "Generate" with no transformation capability can't explain refinement, and "Select" with no explicit criteria can't explain directed evaluation.

### Recommendation: 4-Primitive Model

The 4-primitive model (Attend, Generate, Evaluate, Transform) is the sweet spot:
- Parsimonious enough to be fundamental (4 primitives)
- Expressive enough to compose into all 8 functions (and more)
- Survives tool extinction (these are cognitive primitives)
- Domain-independent (every domain uses all 4)
- Replaces the 8 functions as the architectural primitives

## 7. Whether Creativity Is Better Modeled by Something Other Than Functions

**Verdict: Yes. At least three alternative models are more fundamental.**

### Model A: Creativity as Decisions

A decision-centric model views every creative act as a series of choices under uncertainty:
- What to attend to
- What to generate
- What to keep
- What to change
- When to stop

**Evidence:** Campbell's BVSR, Simonton's chance-configuration theory. Creativity IS decision-making with novelty as a selection criterion.

**Advantage:** Maps cleanly to AI/agent architecture (every agent is a decision-maker). Decisions compose naturally — a decision tree IS the creative process.

### Model B: Creativity as Constraints

Boden's model:
- **Exploratory creativity:** Navigate within existing constraint space
- **Transformational creativity:** Change the constraint space itself

**Evidence:** Stokes (2005) demonstrates that constraint manipulation is the PRIMARY driver of creative breakthroughs. The constraint space IS the creative medium.

**Advantage:** Explains why constraints (budget, time, material, format) often produce MORE creative results than unlimited freedom. This is counterintuitive but well-documented.

### Model C: Creativity as Attention

A model focused on attention allocation:
- Broad attention → exploration, remote associations, novelty
- Narrow attention → refinement, detail, precision
- Attention switching → creative breakthroughs (the "aha" moment)

**Evidence:** Isen (1999) on affect and attention breadth. Martindale (1999) on defocused attention and creativity.

**Advantage:** Explains incubation (attention taken away from the problem), insight (sudden attention restructuring), and the role of environment in creativity.

### Synthesis: The 4-Primitive AGE^T Model

The strongest model integrates all three alternatives into the 4 primitives:

```
Attend (where attention goes)
  → Generate (novelty production)
    → Evaluate (judgment against criteria)
      → Transform (change based on evaluation)
        → loop
```

Decisions are the output of Evaluate.
Constraints are the input to Generate and the criteria for Evaluate.
Attention is the meta-primitive that directs all others.

## 8. Final Verdict: Does v4 Survive Invalidation?

### What Survives

1. The 8 functions are a **usable heuristic** for organizing specialist agents. They map well to observable behaviors in creative production.
2. The intent-centric architecture (Intent → Strategy → Functions) is structurally sound.
3. The tool-agnostic principle (functions survive tool extinction) remains valid — though the list needs adjustment.

### What Does Not Survive

1. **The 8 functions are NOT primitives.** They are compositions of 3-5 deeper primitives (Attend, Generate, Evaluate, Transform).
2. **The list is incomplete.** Missing: Incubation, Play, Affect, Problem Finding, Constraint Transformation.
3. **Some functions collapse.** Evaluate+Refine are inseparable. Realize+Evaluate+Refine form a triplet. Communication and Integration are emergent, not primary.
4. **Cross-domain validity is weak.** The model fits film/TV production well (it was shaped by it) but breaks for music, painting, coding, and improvisation.
5. **Levels of analysis are confused.** "Select" and "Evaluate" are micro-cognitive processes. "Communicate" and "Integrate" are macro-production tasks. Mixing them on the same level creates category errors.

### What v4 Should Become

```
Primitives (4): Attend, Generate, Evaluate, Transform
  ↓ compose into
Composite Functions (5-8): Frame, Explore, Select, Realize, Refine, Integrate, Communicate
  ↓ specialized by
Domain Agents (per creative domain)
  ↓ executed via
Capabilities (tool-agnostic transforms)
  ↓ bound to
Actions + Tools (implementation layer)
```

The primitives are the true universal layer. The composite functions are the organizational layer. The 8-function list needs adjustment:

| Layer | Components | Survives Tool Extinction? | Universal? |
|-------|-----------|--------------------------|------------|
| **4 Primitives** | Attend, Generate, Evaluate, Transform | YES ✓ | YES ✓ |
| **Composite Functions** | Frame, Explore, Select, Realize, [Refine - fold into Evaluate+Transform], [Integrate - emergent], [Communicate - social context] | Mostly ✓ | Partially |
| **Domain Agents** | Film Agent, Music Agent, Code Agent | YES ✓ | N/A — domain-specific |
| **Capabilities** | Tool-agnostic named transforms | YES ✓ | Per domain |

### The Core Failure of v4

v4 organized around functions that are too high-level to be primitives but too low-level to be universal. It inherits the structure of film/TV production and tries to apply it to all creativity. A composer, a novelist, a painter, and an engineer do NOT all use "Frame → Explore → Select → Realize → Evaluate → Refine → Integrate → Communicate" in the same way.

The system should be organized around the 4 primitives, which are universal, and allow the composite functions to emerge from domain-specific usage — not be pre-architected.
