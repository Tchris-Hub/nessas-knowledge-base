# Creative Operating System — Project Vision

## Purpose
This repository is a living knowledge base designed to support the future design, architecture, and implementation of a Creative Operating System — a capability orchestration layer that sits above existing creative software and transforms human imagination into professional creative output.

## Core Thesis
Technical skill should not be the bottleneck in creative production. The bottleneck should be imagination, taste, intent, and creative direction. The system should understand intent and coordinate specialist capabilities — the way a production studio coordinates its artists and craftspeople.

## Philosophy
- Study humans first, AI second.
- Understand how elite creative teams work before designing systems.
- The architecture of human expertise informs the architecture of AI capabilities.
- The system represents coordinated specialists, not one monolithic model.

## Long-Term Vision
A Creative Operating System that:
- Interprets creative intent from natural communication
- Coordinates specialist AI capabilities (story, character, modeling, animation, lighting, VFX, editing, direction, quality review)
- Sits above existing tools (Blender, Unreal Engine, video gen, CAD, future platforms)
- Is modular — each capability works standalone or as part of a pipeline
- Is platform-independent — no single software dependency
- Prioritizes creative direction over technical operation

## Research Focus Areas
1. Capability Architecture — what capabilities are needed, how they relate
2. Specialist Architecture — how real-world creative specialists think and decide
3. Knowledge Architecture — what knowledge each specialist requires
4. Workflow Architecture — how creative work flows through a production pipeline
5. Quality Architecture — how quality is defined, measured, and enforced
6. Feedback Architecture — how creative feedback loops operate
7. Orchestration Architecture — how capabilities are coordinated
8. Tool Integration Architecture — how the system connects to real software

## Research Method
- Primary sources preferred: industry documentation, expert interviews, academic sources, real production workflows
- Every finding tracked to its source with confidence level
- Knowledge gaps made visible, not hidden
- Research topics linked into an interconnected knowledge graph
- A domain is complete only when concepts, roles, workflows, dependencies, sources, open questions, and future research opportunities are all documented

## Guiding Questions
- If an elite production studio could be represented as an intelligent capability network, what would that architecture look like?
- What decisions does each specialist make? What knowledge do they need? How do they communicate with other specialists?
- How does creative intent translate through a pipeline into final output?
- How do feedback and iteration cycles work in professional production?
- What can be automated, what should remain human-directed, and what requires collaboration?
